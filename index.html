Computed field logic reference — dev implementation guide

The fields below are not returned by GA4 or GSC APIs. They must be computed client-side (or in the backend before render). Each formula, threshold rule, and classification logic is defined here precisely so the dev can implement without ambiguity. All threshold values should live in a separate config.json — never hardcoded in render logic.

Week-on-week (WoW) % delta
Derived · GA4 two-dateRange call
// Request both ranges in one API call dateRanges: [ { startDate:"7daysAgo", endDate:"today" }, { startDate:"14daysAgo", endDate:"8daysAgo" } ] // GA4 returns dateRangeName per row delta = ((thisWeek - lastWeek) / lastWeek) * 100 display = (delta > 0 ? "▲ " : "▼ ") + Math.abs(delta).toFixed(1) + "% WoW"
Arrow colour: delta ≥ 0 = green pill. delta < 0 = red pill. Exception: avg position — lower is better so invert the colour logic.
KPI status tag logic
Derived · GA4 + GSC + config.json
function getTag(current, target, type) { if (type === "position") { if (current <= target) return "Healthy"; if (current <= target*1.5) return "Monitor"; return "At risk"; } const r = current / target; if (r >= 1.0) return "On track"; if (r >= 0.8) return "Growing"; if (r >= 0.6) return "Monitor"; if (r >= 0.4) return "Early"; return "Below"; } // Badge colour map: // On track / Healthy → badge-up (green) // Growing / Monitor / Early → badge-warn // Below / At risk → badge-down (red)
Define target values in config.json. Re-run this function on every data refresh — do not cache tag strings.
GSC position badge colour
Derived · GSC position value
function posBadge(pos) { if (pos <= 5) return "badge-up"; // green if (pos <= 10) return "badge-warn"; // amber return "badge-down"; // red } // Zone labels for chart annotations: // 1–3 → "Top rank" // 4–10 → "Opportunity zone" // 11–20 → "Quick-win target"
GA4 returns position as a float. Round to 1 decimal for display. Sort the ranked pages table by position ASC before rendering.
Branded vs non-branded classification
Derived · GSC query dimension (client-side JS)
// Update this list when new brands launch const brandTerms = [ "impacteers", "futurejobs", "xooper", "genrichers" ]; function isBranded(query) { return brandTerms.some(t => query.toLowerCase().includes(t)); } // Aggregate after classification: rows.forEach(r => { const bucket = isBranded(r.query) ? branded : nonBranded; bucket.impressions += r.impressions; bucket.clicks += r.clicks; });
GSC has no native branded field. This split must run client-side. Store brandTerms in config.json for easy updates.
LLM Citation channel — filter logic
Derived · GA4 sessionSource + sessionMedium
// Store in config — update quarterly const llmDomains = [ "chat.openai.com", "perplexity.ai", "gemini.google.com", "claude.ai", "you.com", "copilot.microsoft.com", "bing.com/chat", "kagi.com" ]; // GA4 API filter: { filter: { andGroup: [ { fieldName:"sessionMedium", stringFilter:{ value:"referral" }}, { fieldName:"sessionSource", inListFilter:{ values:llmDomains }} ] } }
GA4 does not have a native LLM channel. This is a custom segment. Run as a separate parallel API call and merge into the channel mix before rendering the donut chart.
Organic only toggle — filter scope
Derived · GA4 sessionMedium + LLM list
// When organic toggle = ON: // Include: sessionMedium === "organic" OR sessionSource in llmDomains // Exclude: sessionMedium === "none" // direct sessionMedium === "paid" sessionMedium === "cpc" sessionMedium === "referral" (unless source in llmDomains) // GA4 dimensionFilter equivalent: { orGroup: [ { fieldName:"sessionMedium", stringFilter:{ value:"organic" }}, { fieldName:"sessionSource", inListFilter:{ values:llmDomains }} ] }
The toggle re-queries GA4 with the combined filter. LLM referrals are included because they represent intent-driven organic discovery — not paid or direct visits.
GJE segment isolation
Derived · GA4 sessionSourceMedium
// GJE is NOT in sessionDefaultChannelGroup // as a separate channel — it merges into // Organic Search. Must be extracted first. // Step 1: Run parallel query filtered to: sessionSourceMedium = "google_jobs_apply / organic" // Step 2: Subtract GJE sessions from // the Organic Search total before // rendering the channel mix donut. // Step 3: Add GJE as its own donut segment.
If you skip this, GJE traffic inflates Organic Search numbers and hides the true organic content performance. Always isolate before grouping.
CTR opportunity score
Derived · GSC clicks, impressions, position
// Filter criteria first: const BENCHMARK_CTR = 0.05; // 5% const MIN_IMPRESSIONS = 500; function oppScore(row) { return row.impressions * (BENCHMARK_CTR - row.ctr); } // Apply filter + sort: quickWins = rows .filter(r => r.position > posMin && r.position <= posMax && r.impressions > MIN_IMPRESSIONS) .map(r => ({...r, score:oppScore(r)})) .sort((a,b) => b.score - a.score);
Score represents clicks left on the table if CTR reached the 5% benchmark. Negative scores (already above 5% CTR) are filtered out — those pages are performing well.
Domain-level data segmentation
Derived · GA4 hostname dimension filter
// When sub-domain filter changes, // re-run GA4 request with filter: const domainMap = { all: null, // no filter main: "www.impacteers.com", futurejobs: "futurejobs.impacteers.com", recruitment: "recruitment.impacteers.com", examprep: "examprep.impacteers.com" }; // Add to GA4 dimensionFilter: { fieldName: "hostname", stringFilter: { matchType: "EXACT", value: domainMap[selectedDomain] } }
All 4 domains must share one GA4 property with cross-domain tracking enabled. The hostname dimension differentiates them. Each filter change triggers a fresh API call — use debounce (300ms) to avoid quota waste.
